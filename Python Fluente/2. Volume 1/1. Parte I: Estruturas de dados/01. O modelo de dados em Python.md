### Um baralho pythĂ´nico

O [Exemplo 1](#exemplo-1-um-baralho-como-uma-sequĂŞncia-de-cartas) ĂŠ simples, mas demonstra as possibilidades que se abrem com a implementaĂ§ĂŁo de apenas dois mĂŠtodos especiais, `__getitem__` e `__len__`.

##### _Exemplo 1. Um baralho como uma sequĂŞncia de cartas_

```python

class FrenchDeck:
    ranks = [str(n) for n in range(2, 11)] + list('JQKA')
    suits = 'spades diamonds clubs hearts'.split()

    def __init__(self):
        self._cards = [Card(rank, suit) for suit in self.suits
                                        for rank in self.ranks]

    def __len__(self):
        return len(self._cards)

    def __getitem__(self, position):
        return self._cards[position]
```

A primeira coisa a notar ĂŠ o uso de `collections.namedtuple` para construir uma classe simples representando cartas individuais. Usamos `namedtuple` para criar classes de objetos que sĂŁo apenas um agrupamento de atributos, sem mĂŠtodos prĂłprios, como um registro de banco de dados. Neste exemplo, a utilizamos para fornecer uma boa representaĂ§ĂŁo textual para as cartas em um baralho, como mostra a sessĂŁo no console:

```python

```

Mas a parte central desse exemplo ĂŠ a classe `FrenchDeck`. Ela ĂŠ curta, mas poderosa. Primeiro, como qualquer coleĂ§ĂŁo padrĂŁo de Python, uma instĂ˘ncia de `FrenchDeck` responde Ă  funĂ§ĂŁo `len()`, devolvendo o nĂşmero de cartas naquele baralho:

- Os usuĂĄrios de suas classes nĂŁo precisam memorizar nomes arbitrĂĄrios de mĂŠtodos para operaĂ§Ăľes comuns ("Como obter o nĂşmero de itens? Ă` .size()`, `.length()` ou outra coisa?")

### Como os mĂŠtodos especiais sĂŁo utilizados

A primeira coisa a saber sobre os mĂŠtodos especiais ĂŠ que eles sĂŁo feitos para serem chamados pelo interpretador Python, e nĂŁo por vocĂŞ. VocĂŞ nĂŁo escreve `my_object.__len__()`. Mas o interpretador pega um atalho quando estĂĄ lidando com um tipo embutido como `list`, `str`, `bytearray`, ou extensĂľes compiladas como os arrays da NumPy. As coleĂ§Ăľes de tamanho variĂĄvel de Python escritas em C incluem uma struct[[5](../../5.%20PostfĂĄcio/footnote.md#L5)] chamada `PyVarObject`, com um campo `ob_size` que registra a quantidade de itens na coleĂ§ĂŁo. EntĂŁo, se `my_object` ĂŠ uma instĂ˘ncia de algum daqueles tipos embutidos, `len(my_object)` devolve diretamente valor do campo `ob_size`, e isso ĂŠ mais rĂĄpido que chamar um mĂŠtodo.

Na maior parte das vezes, a chamada a um mĂŠtodo especial ĂŠ implĂ­cita. Por exemplo, o comando `for i in x:` na verdade gera uma invocaĂ§ĂŁo de `iter(x)`, que por sua vez pode chamar `x.__iter__()` se esse mĂŠtodo estiver disponĂ­vel, ou usar `x.__getitem__()`, como no exemplo do `FrenchDeck`.

Em condiĂ§Ăľes normais, seu cĂłdigo nĂŁo deveria conter muitas chamadas diretas a mĂŠtodos especiais. A menos que vocĂŞ esteja fazendo muita metaprogramaĂ§ĂŁo, implementar mĂŠtodos especiais deve ser mais frequente que invocĂĄ-los explicitamente. O Ăşnico mĂŠtodo especial que ĂŠ chamado frequentemente pelo seu cĂłdigo ĂŠ `__init__`, para invocar a inicializaĂ§ĂŁo da superclasse na implementaĂ§ĂŁo do seu prĂłprio `__init__`.

Geralmente, se vocĂŞ precisa invocar um mĂŠtodo especial, ĂŠ melhor chamar a funĂ§ĂŁo embutida relacionada (por exemplo, `len`, `iter`, `str`, etc.). Essas funĂ§Ăľes chamam o mĂŠtodo especial correspondente, mas tambĂŠm fornecem outros serviĂ§os e para tipos embutidos sĂŁo mais rĂĄpidas que chamadas a mĂŠtodos.

##### _Exemplo 2. Uma classe simples para representar um vetor 2D._

```python

especially in the ``__add__`` and ``__mul__`` methods.

This example is greatly expanded later in the book.

Addition::

    >>> v1 = Vector(2, 4)
    >>> v2 = Vector(2, 1)
    >>> v1 + v2
    Vector(4, 5)

    >>> v = Vector(3, 4)
    >>> abs(v)
    5.0

Scalar multiplication::

    >>> v * 3
    Vector(9, 12)
    >>> abs(v * 3)

"""


import math

class Vector:

    def __init__(self, x=0, y=0):
        self.x = x
        self.y = y

    def __repr__(self):
        return f'Vector({self.x!r}, {self.y!r})'

    def __abs__(self):
        return math.hypot(self.x, self.y)

    def __bool__(self):
        return bool(abs(self))

    def __add__(self, other):
        x = self.x + other.x
        y = self.y + other.y
        return Vector(x, y)

    def __mul__(self, scalar):
        return Vector(self.x * scalar, self.y * scalar)
```

Implementamos cinco mĂŠtodos especiais, alĂŠm do costumeiro `__init__`. Veja que nenhum deles ĂŠ chamado diretamente dentro da classe ou durante seu uso normal, ilustrado pelos doctests. Como mencionado antes, o interpretador Python ĂŠ o Ăşnico usuĂĄrio frequente da maioria dos mĂŠtodos especiais.

O [Exemplo 2](#exemplo-2-uma-classe-simples-para-representar-um-vetor-2d) implementa dois operadores: `+` e `*`, para demonstrar o uso bĂĄsico de `__add__` e `__mul__`. No dois casos, os mĂŠtodos criam e devolvem uma nova instĂ˘ncia de `Vector`, e nĂŁo modificam nenhum dos operandos: self e other sĂŁo apenas lidos. Esse ĂŠ o comportamento esperado de operadores infixos: criar novos objetos e nĂŁo tocar em seus operandos. Vou falar mais sobre esse tĂłpico no [CapĂ­tulo 16](../../3.%20Volume%202/1.%20Parte%20III:%20Classes%20e%20Protocolos/06.%20Sobrecarga%20de%20operadores.md).

#### RepresentaĂ§ĂŁo como string

O mĂŠtodo especial `__repr__` ĂŠ chamado pela funĂ§ĂŁo embutida `repr` para obter a representaĂ§ĂŁo do objeto como uma string, para depuraĂ§ĂŁo. Sem um `__repr__` customizado, o console de Python mostraria uma instĂ˘ncia de `Vector` como `<Vector object at 0x10e100070>`.

O console iterativo e o depurador chamam `repr` para exibir o resultado das expressĂľes. O `repr` tambĂŠm ĂŠ usado:

- Pelo marcador posicional `%r` na formataĂ§ĂŁo clĂĄssica com o operador `%`. Ex.: `'%r' % my_obj`
